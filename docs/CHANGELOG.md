## Convenções de Versioning

- **Major (X.0.0):** Lançamentos principais com novos recursos significativos
- **Minor (0.X.0):** Novos recursos e melhorias
- **Patch (0.0.X):** Correções de bugs e ajustes menores

## Como Contribuir

Quando fizer alterações significativas:

1. Atualize a versão em `package.json` (se aplicável)
2. Adicione um nova seção em [Não Lançado] com suas mudanças
3. Organize as mudanças em:
   - **Added** - para novas funcionalidades
   - **Changed** - para mudanças em funcionalidades existentes
   - **Deprecated** - para funcionalidades que serão removidas em breve
   - **Removed** - para funcionalidades removidas
   - **Fixed** - para correções de bugs
   - **Security** - para correções de segurança
